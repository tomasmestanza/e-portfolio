Compare the effectiveness of each tool in defining and identifying code quality. What can you conclude about the effectiveness of each approach?

It is essential to mention that the checks of all tools can be set, and it will depend on the rules that are shown after the test. This test has been done with the default settings.

Pylint seems to include most of the functions in Pyflake, highlighting included libraries that are to be used. Pylint provides model details and can be very descriptive. Some of the Pylint recommendations are overly strict, as parameters defined as one character are highlighted as not compliant with the snake case convention, for example.

Pylint can also highlight some of the same issues as Pycodestyle. In terms of spaces, it gives suggestions about unnecessary brackets in if statements. Also, unnecessary semi-colons are reported with Pylint.

Pylint also reports issues with the doctoring and comments; this is a very useful feature.

To summarise, Pylint is a complete tool that, in a way, includes the functionality of all other tools. Although in some cases, Pylint gives more details, in others, it is very generic, pointing at the issue. For example, it doesnâ€™t highlight the docstring issue in the __init__ method, but this is reported with Pydocstyle. I still believe that in the absence of other tools, Pylint would be my first choice.
