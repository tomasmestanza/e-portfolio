What is an Ontology?

What do you understand about the ontology that has been presented for your reading this week? Could you attempt to define an ontology that would be relevant to the system that you are designing for the summative assessment?

Becker et al. (2002) define ontology in software as the specification of an abstract model of a situation happening in the world, including the concepts relevant to this specific situation or domain. So, being an abstraction, a human has to assume the point of view of a computer to define the ontology of software.

The ontology presented in the article by Arnaut et al. (2010) aims to overtake the restrictions of pre-existing ontologies that can only be used at running time and use web services technology for search and recovery. The proposed ontology is meant to be used in the development phase of the SDLC of SOA (Software-oriented architecture).

With the previous concepts in mind, the following ontology represents the concepts within the domain of the IIS system (Astronaut health Monitoring) we are working on.

Exploring a simple Python shell

Review the blogs at Praka (2018) and Szabo (n.d.) and then create a CLI/ shell that implements the following:

When you enter the command LIST it lists the contents of the current directory

  • The ADD command will add the following two numbers together and provide the result
  • The HELP command provides a list of commands available
  • The EXIT command exits the shell
  • Add suitable comments to your code and add the program to your e-portfolio. Be prepared to demonstrate it in the seminar session next week.

Run the shell you have created, try a few commands and then answer the questions below. Be prepared to discuss your answers in the seminar.

Answer

Following the example presented by Szabo (N.D.), we have expanded the functionality to include a functioning ADD command and a LIST command.

from cmd import Cmd
import os

class SSDPrompt(Cmd):
   prompt = 'pb> '
   intro = "Welcome! Type ? to list commands"

   def do_exit(self, inp):
      '''exit the application.'''
      print("Bye")
      return True

   def help_exit(self):
      print('exit the application. Shorthand: x q Ctrl-D.')

   def do_list(self, inp):
      '''list the content of the current directory.'''
      cwd = os.getcwd()
      dir_list = os.listdir(cwd)
      print("Files and directories in '", cwd, "' :")
      # prints all files and folders
      print(dir_list)

   def do_add(self, inp):
      '''adds two numbers.'''
      print("adding '{}'".format(inp))

      num=inp.split()
      print ('addition:',int(num[0])+int(num[1]))

   def help_add(self):
      print("Add a new entry to the system.")

   def default(self, inp):
      '''support for when the inputed command doesn not exist.'''
      if inp == 'x' or inp == 'q':
         return self.do_exit(inp)

      print("Default: {}".format(inp))

   do_EOF = do_exit
   help_EOF = help_exit

if __name__ == '__main__':
SSDPrompt().cmdloop()


Vulnerability: Command injection
The inputs are not sanitised which can allow potential attackers to execute malicious code.

Preventions:
1. In general, the use of python 3 will cover a good number of vulnerabilities presented in the previous version (v2).

2. Input validation and sanitisation play an important role to prevent different types of injection. In this specific example, all inputs to the command methods should be validated and sanitised before executing the commands.
