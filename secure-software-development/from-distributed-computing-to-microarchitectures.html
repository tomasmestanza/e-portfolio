Collaborative Discussion 2: Cryptography case study: TrueCrypt

Summary post

In my initial post, I pointed to the importance of context/time when analysing a solution. Placing the TrueCrypt in the right context is key to understanding its advantages and disadvantages and why it was or was not an acceptable encryption solution.

I would like to insist on the fact that the study by Junestam & Guigo (2014) found no vulnerability categorised as critical, and therefore looking at other available solutions was very important to choose or not TrueCrypt as a solution.

One thing was clear, and it was that TrueCrypt could not be used these days, and its use should not be recommended. In my view, the use of such an old solution and not a more intelligent approach available these days can not be justified.

Faceted Data

Read Schmitz et al (2016) article about faceted data.

Question 1
Do you think this is a good approach to protect systems from data leakage? What are the pros and cons?

Answer

The solution proposed in the article offers a solution to maintain the secure flow of information within a system. Applications without such control are more likely to leak sensitive information.

Using Haskell language, defined by (Haskell, N.D.) as an advanced purely functional language that allows the production of robust and correct software systems, the solution is based on faceted values and imperative programming through monads.

Although the solution appears to be complex, assessing the cost/benefit is critical to adopt such a solution. If the workflow within a system is not secure, the information is in a constant state of vulnerability.

Question 2
Create a basic outline design of how you would create such a system in Python.

Answer

  • 1. Identify the risks and the levels of acceptability.
  • 2. Design an architecture that allows the implementation of security in every stage of the SDLC.
  • 3. Write the code taking into account the security outlined in the design.
  • 4. Apply security tests and test the deployment.
  • 5. Apply necessary patches as a measure of security maintenance.
